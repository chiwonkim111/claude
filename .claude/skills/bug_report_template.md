---

검수 중 버그 또는 이슈를 발견하면 아래 템플릿과 분류 기준을 따라 기록합니다.

## 이슈 심각도 분류

| 등급 | 기호 | 기준 | 조치 기한 |
|------|------|------|----------|
| **Critical** | 🔴 | 서비스 운영 불가 / 데이터 손실 / 보안 취약점 | 즉시 |
| **High** | 🟠 | 핵심 기능 동작 불가 / 기획 명세 미구현 | 이번 스프린트 |
| **Medium** | 🟡 | 부분 동작 / UX 저하 / 성능 문제 | 다음 스프린트 |
| **Low** | 🔵 | 오탈자 / 스타일 불일치 / 부가 기능 오류 | 여유 시 |
| **Info** | ⚪ | 개선 제안 / 리팩토링 권고 | 검토 후 결정 |

---

## 버그 리포트 템플릿

```markdown
### [BUG-{번호}] {이슈 제목}

- **심각도**: 🔴 Critical / 🟠 High / 🟡 Medium / 🔵 Low / ⚪ Info
- **분류**: 기능 오류 / 기획 불일치 / 보안 / 성능 / 접근성 / 코드 품질
- **발견 위치**: `파일경로:라인번호` 또는 `API 엔드포인트`
- **관련 요구사항**: FR-{번호} 또는 US-{번호}

#### 현상 (What)
> 실제로 어떤 문제가 발생하는지 설명합니다.

#### 기대 동작 (Expected)
> docs/PLAN.md 또는 docs/ARCHITECTURE.md 기준으로 어떻게 동작해야 하는지 설명합니다.

#### 실제 동작 (Actual)
> 현재 코드가 어떻게 동작하는지 설명합니다.

#### 재현 방법 (Steps)
1. ...
2. ...
3. ...

#### 근거 코드
```코드언어
// 문제가 되는 코드 스니펫
```

#### 해결 방안 제안
> 어떻게 수정하면 되는지 간략히 제안합니다.
```

## 기획 불일치 리포트 템플릿

기능이 기획서와 다르게 구현된 경우 사용합니다.

```markdown
### [MISMATCH-{번호}] {기능명} 기획-구현 불일치

- **심각도**: 🟠 High
- **분류**: 기획 불일치
- **관련 요구사항**: FR-{번호}

| 항목 | 기획서 내용 | 실제 구현 |
|------|------------|----------|
| 동작 방식 | | |
| 입력 형식 | | |
| 출력 형식 | | |
| 예외 처리 | | |

#### 수정 방향
> docs/PLAN.md를 기준으로 어떻게 수정해야 하는지 설명합니다.
```

## 미구현 기능 리포트 템플릿

기획서에 있지만 코드가 없는 기능을 기록할 때 사용합니다.

```markdown
### [MISSING-{번호}] {기능명} 미구현

- **심각도**: 🟠 High / 🔴 Critical (MVP 기능인 경우)
- **분류**: 미구현
- **관련 요구사항**: FR-{번호}

#### 기획 내용
> docs/PLAN.md에서 해당 기능 명세를 인용합니다.

#### 구현 필요 항목
- [ ] 항목 1
- [ ] 항목 2

#### 예상 구현 위치
> 어떤 파일/모듈에 구현해야 하는지 docs/ARCHITECTURE.md 기준으로 안내합니다.
```

## QA 리포트 이슈 섹션 작성 예시

```markdown
## 이슈 목록

### 🔴 Critical

#### [BUG-001] 로그인 실패 시 비밀번호 평문이 로그에 출력됨
- **발견 위치**: `src/modules/auth/auth.service.ts:42`
- **현상**: 로그인 실패 시 `console.log(password)`로 평문 비밀번호가 서버 로그에 기록됨
- **기대 동작**: 민감 정보는 로그에 절대 출력하지 않아야 함
- **해결 방안**: `console.log(password)` 라인 즉시 제거

---

### 🟠 High

#### [MISMATCH-001] 사용자 목록 API 페이지네이션 미적용
- **발견 위치**: `GET /api/v1/users`
- **관련 요구사항**: FR-003
- **기획**: 페이지당 20개, 전체 수 반환
- **구현**: 전체 목록을 한 번에 반환 (페이지네이션 없음)
- **해결 방안**: `page`, `limit` 쿼리 파라미터 추가 및 응답에 `meta.pagination` 포함

---

### ⚪ Info

#### [INFO-001] UserService 메서드 길이 초과
- **발견 위치**: `src/modules/users/users.service.ts:78`
- **현상**: `updateUser` 메서드가 34줄로 20줄 제한 초과
- **해결 방안**: 내부 로직을 `validateUpdateInput()`, `applyUpdate()` 등으로 분리 권고
```
