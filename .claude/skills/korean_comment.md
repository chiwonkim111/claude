지금부터 작성하거나 수정하는 모든 코드에 한국어 주석을 작성합니다.

## 한국어 주석 작성 규칙

### 1. 주석 대상

아래 항목에는 반드시 한국어 주석을 작성합니다:

- **파일 상단**: 파일의 역할과 책임을 1~3줄로 설명
- **클래스 / 인터페이스**: 해당 클래스의 목적과 사용 맥락 설명
- **함수 / 메서드**: 기능 설명, 매개변수, 반환값, 예외 상황
- **복잡한 로직**: 한눈에 이해하기 어려운 알고리즘이나 비즈니스 규칙
- **상수 / 열거형**: 값의 의미와 사용 목적
- **타입 / 인터페이스 필드**: 각 필드의 의미와 제약조건

### 2. 주석 형식

**파일 상단 주석:**
```typescript
/**
 * 사용자 인증 관련 비즈니스 로직을 처리하는 서비스
 * - 로그인, 로그아웃, 토큰 갱신 기능을 담당
 * - 인증 실패 시 커스텀 예외를 발생시킴
 */
```

**함수/메서드 주석:**
```typescript
/**
 * 이메일과 비밀번호로 사용자를 인증하고 JWT 토큰을 반환합니다.
 * @param email - 사용자 이메일 주소
 * @param password - 평문 비밀번호 (내부에서 해시 비교)
 * @returns 액세스 토큰과 리프레시 토큰
 * @throws InvalidCredentialsException - 이메일 또는 비밀번호가 일치하지 않을 때
 */
```

**인라인 주석 (복잡한 로직):**
```typescript
// 비밀번호 재시도 횟수가 5회를 초과하면 계정을 잠금 처리
if (user.failedLoginCount >= MAX_LOGIN_ATTEMPTS) {
  await this.lockAccount(user.id);
}
```

**상수/열거형 주석:**
```typescript
// 로그인 최대 재시도 횟수 (초과 시 계정 잠금)
const MAX_LOGIN_ATTEMPTS = 5;

enum UserStatus {
  ACTIVE = 'active',     // 정상 활성 상태
  SUSPENDED = 'suspended', // 관리자에 의해 정지된 상태
  DELETED = 'deleted',   // 소프트 삭제된 상태
}
```

**타입/인터페이스 필드 주석:**
```typescript
interface CreateUserDto {
  /** 사용자 이메일 (로그인 ID로 사용, 중복 불가) */
  email: string;
  /** 평문 비밀번호 (서비스 레이어에서 해시 처리) */
  password: string;
  /** 닉네임 (2~20자, 특수문자 불가) */
  nickname: string;
}
```

### 3. 언어 혼용 원칙

| 항목 | 언어 | 예시 |
|------|------|------|
| 주석 내용 | 한국어 | `// 사용자 목록을 페이지네이션으로 조회` |
| 변수명 / 함수명 | 영어 | `getUserList`, `accessToken` |
| 클래스명 | 영어 | `UserService`, `AuthController` |
| 파일명 | 영어 | `user.service.ts` |
| 에러 메시지 (사용자 노출) | 한국어 | `'이메일 형식이 올바르지 않습니다.'` |
| 에러 메시지 (내부 코드) | 영어 | `INVALID_EMAIL_FORMAT` |

### 4. 금지 사항

- `// TODO`, `// FIXME` 등 영어 태그는 한국어로 대체합니다.
  - `// TODO:` → `// 할 일:`
  - `// FIXME:` → `// 수정 필요:`
  - `// HACK:` → `// 임시 처리:`
  - `// NOTE:` → `// 참고:`
- 코드와 동일한 내용을 반복하는 무의미한 주석은 작성하지 않습니다.
  ```typescript
  // 나쁜 예: 코드를 그대로 반복
  // i를 1 증가시킴
  i++;

  // 좋은 예: 의도와 이유를 설명
  // 다음 페이지로 이동하기 위해 페이지 번호를 증가시킴
  currentPage++;
  ```
- 주석이 코드보다 길어지는 경우 함수를 분리하는 것을 우선 고려합니다.

### 5. 기존 코드 수정 시

기존 코드를 수정할 때는 아래 규칙을 따릅니다:

- 수정한 함수나 로직에는 반드시 한국어 주석을 추가하거나 갱신합니다.
- 기존 영어 주석이 있다면 한국어로 교체합니다.
- 주석이 없는 함수를 수정하는 경우 주석을 새로 추가합니다.
