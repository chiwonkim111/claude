```markdown
## 4. 사용자 여정 (User Journey)

### 핵심 시나리오: [시나리오명]

```
[진입] → [탐색] → [핵심 행동] → [결과 확인] → [이탈/재방문]
  |          |           |              |                |
(채널)    (화면)       (액션)         (피드백)        (트리거)

```
단계별 감정 곡선:
- 진입: 😐 기대
- 탐색: 🤔 탐색
- 핵심 행동: 😊 만족 / 😤 불편
- 결과 확인: 😄 성취

### 5.2 기능 상세 명세

#### FR-001. [기능명]

- **설명**: 기능에 대한 상세 설명
```

## 작성 규칙

- 모든 기능에는 고유 ID를 부여합니다 (`FR-001`, `US-001` 등).
- 완료 조건(Acceptance Criteria)은 테스트 가능한 형태로 작성합니다.
- "빠르게", "쉽게" 같은 모호한 표현 대신 측정 가능한 수치를 사용합니다.
- 미결 사항은 반드시 별도 섹션에 기록하고 방치하지 않습니다.
